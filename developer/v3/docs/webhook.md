# Общие сведения

**Webhook** — механизм отправки уведомлений об определённых событиях в CRM (в основном о действиях пользователей) во внешние приложениях. Интересен он может быть, в первую очередь, для разработчиков и интеграторов, т.к. дополняет систему событий (Events), RestAPI и дает возможности двухстороннего обмена данными именно тогда, когда они изменяются.

![](https://salesman.pro/docs.img/docs/webhook_list.png)

## Как работают WebHook в SalesMan?

Вебхуки привязываются к событиям, которые срабатывают на определенные действия сотрудников в автоматическом режиме, т.е. пользователям не надо совершать каких-либо дополнительных действий.

Для подключения своего скрипта принимающего данные из CRM, необходимо добавить новый хук, выбрать событие, на которое он будет срабатывать и сохранить.

![](https://salesman.pro/docs.img/docs/webhook_add.png)

При срабатывании хука система запишет в лог отправленные параметры и ответ от вашего скрипта. Отправка данных происходит с помощью cURL.

![](https://salesman.pro/docs.img/docs/webhook_log.png)

## Где можно использовать
Вебхуки можно использовать для написания плагинов и интеграции со сторонними системами:

*     оперативного оповещения внешних систем об изменениях в конкретной записи
*     расширение логики CRM с помощью постобработки
*     написания плагинов
